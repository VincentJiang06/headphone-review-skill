# headphone-review-skill

[English Version](./README.md)

`headphone-review` 是一个用于输出买家视角耳机评测的 Codex skill，适用于耳机、IEM、耳塞和 TWS 产品。

它的目标不是机械搬运参数表，而是回答真正和购买有关的问题：

- 这款耳机大致是什么声音
- 它更适合什么人
- 它不适合什么人
- 它真正的强项和代价分别是什么

- 在入口层强制要求用选项式 `AskUserQuestion` 询问语言和风格，不再允许模糊自由提问

## 这个 Skill 会做什么

在开始写评测之前，这个 skill 会通过 `AskUserQuestion` 先询问两件事：

1. 你想要中文还是英文输出
2. 你想要更客观的描述，还是更情绪化、抽象一些的描述

确认后，它会：

- 锁定具体型号、代际与形态
- 搜集官方信息和多语言评测共识
- 区分已确认事实与跨来源共识
- 把声音特征翻译成购买建议
- 输出纯 Markdown 的长文式评测

## 输出特点

- 开头先给出语言、描述风格、两句话短评、风格关键词和置信度
- 正文扩展为长文，但整体长度通常收敛在 3000 词级别以内
- 专门强调“分歧点”章节，但要求更短、更直白、更清楚
- 新增独立的人类化约束，专门压制常见 AI 味句式和节奏
- 优先保障中文输出质量
- 对型号歧义和证据不足会明确标注假设和低置信度

## 输出协议

- 先问语言：`中文` 或 `English`
- 再问描述模式：`客观` 或 `情绪化抽象`
- 开头先输出短标签，再进入正文
- 中文模式下，两句话短评控制在 200 字以内
- 分歧点要求短，但必须具体到会影响购买判断
- 文末不再堆信息来源和参考来源

## 适用场景

- 快速判断某款耳机是否适合自己
- 把零散的多语言评测整理成一份买家视角结论
- 测试中文短文式音频评测输出
- 对多个型号进行调音与人群定位比较

## 写作约束

- 不要让文章滑回通用助手口吻
- 避免高频使用对照模板和作文式转场词
- 所有关键声音判断都要落回实际听感后果
- 分歧点应该像编辑注，不像第二篇文章

## 使用方式

在 Codex 中调用：

```text
$headphone-review
```

然后输入型号。skill 会先询问语言和描述风格，再继续生成评测。
